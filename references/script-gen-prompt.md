# 脚本生成提示词(把源材料 → 分幕脚本 config)

这份文件把"第 1 步:写脚本"从靠判断变成**照提示词产出**。用途:

- **Claude 触发 skill 时**:读本文件,按下面的「生成程序」把源材料写成 `config.json` 的 `scenes`。
- **stoneyang / 任何 AI 独立用**:把「PROMPT」整段 + 你的源材料喂给任一有能力的大模型,它会吐出 scenes JSON,再套进 config 顶层即可渲染。

配套必读:`scene-types.md`(10 种幕的字段)、`config-schema.md`(顶层结构)、`style-and-assets.md`(主题/音色/BGM)。

---

## 叙事弧库(选一个当骨架,可魔改)

弧只是起点,**内容永远服从源材料**;缺的节拍删掉、需要的节拍加上。

| 弧 | 适用 | 节拍(每拍 = 1~2 幕) |
|---|---|---|
| **产品/融资 pitch** | 给投资人/合作方看产品 | 钩子/市场 → 痛点(现状不行) → 解法(产品核心) → 差异化/护城河 → 结果/真实进展 → 规模化/CTA |
| **功能讲解** | 讲清一个产品/功能怎么用 | 是什么(一句话) → 为谁解决什么痛 → 怎么用(步骤/原理) → 效果 → CTA |
| **案例/客户故事** | 用一个真实案例证明价值 | 主角困境 → 转折(用了什么) → 怎么做(拆解) → 结果跃迁 → 可复制/规模 |
| **成果/数据发布** | 秀成绩、发布数据 | 标题 → 核心数字 → 拆解归因 → 意义/愿景 |

## 节拍 → 幕类型 映射

| 你要表达 | 用哪种幕 |
|---|---|
| 钩子 / 章节标题 / 收尾 CTA | `title` |
| 用户原话 / 痛点自述 / 证言 | `quote` |
| 通用方案的空洞 / 常见误区(可盖章否定) | `bullets`(+`stamp`) |
| 方法论 / 步骤 / 思维链 / 多维拆解 | `breakdown` |
| 风险机会评分 / 指标打分 | `score` |
| 正反权衡 / 消除疑虑 / 自证严谨 | `debate` |
| 功能 / 特性 / 定价档 / 并列依据 | `cards` |
| 多 agent / 多模块编排分工 | `dispatch` |
| 结果数字 / 增长 / before→after | `stats`(可 `trend`) |
| 规模化 / 品类铺开 / 品牌落板 | `scale` |

## 生成程序(6 步)

1. **定位**:一句话写清「卖什么 / 给谁看 / 要什么调性」。拿不准就先问用户,别猜。
2. **选弧**:从弧库挑一个最贴的当骨架(默认「产品/融资 pitch」)。
3. **排幕**:把源材料的要点按弧铺成 6~9 幕,每幕定 `type`。控制总时长 60~90 秒 ≈ 旁白总字数 250~380 字(中文约 4~5 字/秒)。
4. **填内容**:每幕按 `scene-types.md` 的字段填,内容**只用源材料里的真实事实/数字**。
5. **写旁白**(`vo`):见下方旁白规则。
6. **自检**:过一遍下方清单,再输出。

## 旁白(vo)规则

- **一幕一句主线**:每幕旁白只推进一个意思,20~55 字为宜。
- **口语、短句**:念出来顺口,像人说话,不像书面语。("很多品牌已经在 AI 里隐形了,却不知道" > "众多品牌于生成式引擎中之可见性缺失而不自知")。
- **和屏上字互补,不照念**:屏幕给标题/数字/要点,旁白给"为什么/意味着什么",两者别重复。
- **数字读出来**:"68%" 写成旁白念 "六成八","¥30亿" 念 "三十亿"。
- **不用套路词**:禁 震惊/必看/爆款/颠覆/10x/史上最(呼应 stoneyang brand voice:真实、不夸、给希望)。

## 真实性 & voice 红线(来自 stoneyang CLAUDE.md)

- **数字/案例/事实只从源材料来**;源材料没有、算不出的,标 `[待确认]` 不臆造。
- **概念演示 / 计划未落地** → 设 `watermark`(如"产品方案·案例已脱敏"),旁白不夸兑现("规划中"别说成"已实现")。
- **命名 / 核心主张 / 归类**:先给候选让 stoneyang 拍板,别在脚本里替他定死品牌名或大主张。
- 人名/敏感数据按需脱敏("张总"→"某连锁餐饮老板")。

## 输出前自检清单

- [ ] 每幕只讲一件事?(一幕塞两个意思就拆成两幕)
- [ ] 每个 `type` 都在 10 种库里?字段齐不齐?
- [ ] 旁白口语、无套路词、和屏上字不重复?
- [ ] 所有数字/事实都能追到源材料?没有的标了 `[待确认]`?
- [ ] 总时长 60~90 秒(旁白总字数 ≤ ~380)?超了砍幕或砍字。
- [ ] 结尾不套路(别每次都"发光球阵+落板";CTA/定价卡/一句话主张都行)?
- [ ] 节点/条目没超标(breakdown≤6、cards≤4、bullets≤5)?
- [ ] **反 AI 味**:没用 emoji 当图标(cards/evidence 默认不填 `icon`)?accent 主色没到处涂(一屏≤2 处强调)?没有多余装饰?(详见 SKILL.md「视觉负面准则」)

## 输出格式

直接产出可用的 `scenes` 数组(或整份 config)。示例骨架:

```json
{
  "brand": "<顶标,留空不显>",
  "watermark": "<角标或空>",
  "aspect": "16:9",
  "theme": "<warm-dark|midnight|clean-light>",
  "voice": "alex", "speed": 0.95,
  "bgm": "<inspired|the_complex|none>", "bgmVolume": 0.055, "sfx": true,
  "scenes": [ /* 6~9 幕,见下 */ ]
}
```

## ========== PROMPT(可整段复制给任一大模型)==========

你是产品演示视频的脚本编剧。我给你一段**源材料**和**目标**,你产出一支 60~90 秒短视频的**分幕脚本**,输出为严格的 JSON(scenes 数组)。

**源材料**:{在此粘贴 PRD / 案例 / 数据 / 一段产品介绍}
**目标**:{卖什么 · 给谁看 · 要什么调性;如「给投资人看的 GEO SaaS 产品 pitch,专业但接地气」}
**叙事弧**:{可指定,或写"你来选最合适的"}

要求:
1. 从下面 10 种幕类型里挑着拼 6~9 幕:`title`(标题/CTA)、`quote`(原话)、`bullets`(要点+可盖章)、`breakdown`(结论+分层节点)、`score`(打分条)、`debate`(红蓝对抗+裁决+依据卡)、`cards`(卡片)、`dispatch`(中枢派单)、`stats`(数字+可选趋势线)、`scale`(规模网格+落板)。
2. 每幕给该类型对应字段(不确定字段就参照:title=kicker/title/subtitle/tagline;quote=label/text;bullets=label/items/stamp{text};breakdown=headingLabel/heading/nodes[{k,v}]/可选persona;score=label/bars[{label,score,color}];debate=label/sides[{dot,tag,color,text}]/verdict{tag,text}/evidence[{icon,label,text}];cards=label/cards[{icon,label,text}];dispatch=persona/note/agents[{name,items}];stats=kicker/title/trend/stats[{big,cap}];scale=items[{label}]/lines/brandEnd/brandSub),并给一句 `vo`(旁白)。
3. **内容只用源材料里的真实事实与数字**;源材料没有的标 `[待确认]`,绝不编造。
4. **旁白**:一幕一句主线,20~55 字,口语短句,和屏上字互补不照念,数字用中文念法;禁用 震惊/必看/爆款/颠覆/10x 等夸张词。
5. 概念演示或未落地的内容,提醒设 watermark、旁白不夸兑现。品牌名/核心主张若源材料没定,列 2~3 个候选让我选,别自己定死。
6. 每幕只讲一件事;breakdown 节点≤6、cards≤4、bullets≤5;旁白总字数 ≤ 380(控制在 90 秒内)。
6b. **反 AI 味**:cards/evidence **默认不要填 `icon`**(emoji 图标是 AI 味头号信号);要视觉锚点用序号或纯文字 label。强调主色稀缺用(一屏≤2 处),不要每个 label/节点都上色。宁可放大字号,不堆颜色和图标。
7. 先给一句「定位 + 选用的叙事弧 + 每幕一句话概览」,再输出 JSON。

现在开始。

## 一个 mini 示例(输入 → 输出)

**输入**:源材料="一个把长视频自动剪成竖屏切片的工具,接入抖音API,给自媒体用";目标="给自媒体用户的功能讲解,明快"。

**输出概览**:定位=自媒体人的自动切片工具/功能讲解弧。
1. `title` 钩子:"一条长视频,手剪 10 条切片要一下午"
2. `breakdown` 怎么用:上传→AI 找高光→自动竖屏裁切→加字幕→一键发抖音(4~5 节点)
3. `cards` 亮点:自动找高光 / 竖屏智能构图 / 字幕自动生成 / 一键分发
4. `stats` 效果:一下午→5 分钟 / ×10 产量(标 `[待确认]` 若无实测)
5. `title` CTA:"把剪辑的时间,还给创作"
（每幕配一句口语 vo)
