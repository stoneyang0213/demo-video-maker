# 幕类型库(写 config.scenes 前必读)

每个幕对象 = `{"type": "<类型>", ...字段..., "vo": "旁白(可选)", "dur": 秒(无 vo 时用), "w": 内容列宽px(可选覆盖), "endPad": 尾部留白帧(可选)}`。

配色字段(bar.color / stamp.color / side.color 等)可填**主题色名**(`accent` `danger` `blue` `redd` `ok` `soft`)或直接 hex(`"#FF6A00"`)。

---

## title — 标题/封面/落板
开场亮相或章节标题。
| 字段 | 说明 |
|---|---|
| `kicker` | 顶部小标(mono 字体),如 `"CASE · AutoClip 切片工具"` |
| `kickerColor` | kicker 颜色名,默认 accent |
| `title` | 主标题 |
| `subtitle` | 副标题(灰) |
| `tagline` | 底部一句话 |
| `big` | true=更大字号 |
**用在**:开篇困境亮相、产品名亮相、章节过渡。

## quote — 引用/原话(打字机)
用户原话、痛点、证言,逐字打出来。
| 字段 | 说明 |
|---|---|
| `label` | 卡片上方小标,如 `"学员的问题"` |
| `text` | 引用正文(打字机效果) |
| `title` | (可选)卡片上方大标题 |
| `cps` | 打字速度(字/秒),默认 20 |
**用在**:学员/客户原话、痛点自述、证言。

## bullets — 要点列表 + 可选盖章
罗列几条(常用于"通用做法/泛泛而谈"),可盖一个红章否定它。
| 字段 | 说明 |
|---|---|
| `label` | 上方小标,如 `"通用 AI"` |
| `items` | 字符串数组,逐条淡入 |
| `stamp` | (可选)`{"text":"正确的废话","at":150,"color":"danger"}` 盖章 |
**用在**:对比的"上半"(通用方案的空洞)、常见误区、被否定的清单。

## breakdown — 结论 + 分层节点(思维链/五维/步骤)
先抛一句结论,再一层层点亮节点(每个 4-6 个最佳)。最出彩的"讲道理"幕。
| 字段 | 说明 |
|---|---|
| `persona` | (可选)人格锚标签,如 `"AI 操盘手"`,显示发光处理核在左 |
| `headingLabel` | 结论上方小标,如 `"综合结论"` |
| `heading` | 结论句(纯文本) |
| `nodes` | `[{"k":"赛道","v":"一句话展开"}, ...]`,逐个点亮 |
| `step` | 节点间隔帧,默认 70(想更快就调小) |
**用在**:方法论拆解、推理链、分步骤、多维分析。

## score — 打分条
量化判断,条形从 0 增长到分值。
| 字段 | 说明 |
|---|---|
| `label` | 上方小标,如 `"卡点 / 机会 · 打分"` |
| `bars` | `[{"label":"...","score":8.5,"color":"redd"}, ...]`(score 0~10) |
**用在**:风险/机会评分、指标对比、能力雷达的线性版。

## debate — 红蓝对抗 + 裁决 + 依据卡
自己正反辩论再裁决,最像"真思考"的一幕。
| 字段 | 说明 |
|---|---|
| `label` | 小标,如 `"红蓝对抗"` |
| `sides` | `[{"dot":"🔵","tag":"蓝方","color":"blue","text":"..."}, ...]` |
| `verdict` | `{"tag":"裁决","text":"..."}`(可选 `dot`,默认 ⚖️) |
| `verdictAt` | 裁决出现帧,默认 215 |
| `evidence` | (可选)`[{"icon":"📎","label":"案例库","text":"..."}, ...]` 依据卡溯源 |
| `evidenceAt` | 依据卡起始帧,默认 260 |
**用在**:权衡取舍、正反论证、agent 自证严谨、消除疑虑。

## cards — 卡片行/列(功能/特性/依据)
并排或纵列的卡片。
| 字段 | 说明 |
|---|---|
| `label` | 上方小标 |
| `cards` | `[{"icon":"⚡","label":"标签","text":"正文","big":false}, ...]` |
| `column` | true=纵向排列(默认横向) |
**用在**:功能三宫格、特性罗列、依据并列、对比项。

## dispatch — 中枢派单(agent 团队/分发)
一个人格中枢向下派发到若干模块,各出可交付物。
| 字段 | 说明 |
|---|---|
| `persona` | 中枢人格标签,如 `"AI 操盘手"` |
| `note` | 中枢下方一句说明 |
| `agents` | `[{"name":"渠道 Agent","items":["交付物1","交付物2"]}, ...]` |
**用在**:多 agent 编排、系统模块分工、"它还帮你干了这些活"。

## stats — 结果数字 + 可选趋势线
大数字 + 增长曲线,结果幕。
| 字段 | 说明 |
|---|---|
| `kicker` | 顶部小标,如 `"RESULT · 一年后"` |
| `title` | 标题 |
| `trend` | true=显示上扬绿线 |
| `stats` | `[{"big":"×2","cap":"海外营收 double"}, ...]` |
**用在**:成果数字、增长、before→after 跃迁、KPI。

## scale — 规模化网格 + 落板
一堆节点铺开 + 两行主张 + 品牌落板,收尾。
| 字段 | 说明 |
|---|---|
| `items` | `[{"label":"密封罐"}, ...]` 网格节点(可省) |
| `columns` | 网格列数,默认 4 |
| `lines` | 两三行大字主张(最后一行自动强调色) |
| `brandEnd` | 落板品牌字标,如 `"KROX AI"` |
| `brandSub` | 品牌下方一句 |
| `endAt` | 落板出现帧,默认 150 |
**用在**:规模化叙事、"一个人→一千个"、品牌收尾、行动号召。

---

## 组合建议
- **融资/产品演示强弧**:title(困境)→ quote(痛点)→ bullets(通用方案空洞+盖章)→ breakdown(我们的拆解)→ score+debate(严谨)→ dispatch(能落地)→ stats(结果)→ scale(规模+落板)。
- **功能讲解**:title → cards(功能)→ breakdown(工作原理)→ stats(效果)→ title(CTA)。
- **数据发布**:title → stats → cards(拆解)→ scale(愿景)。
- 不确定就先少几幕跑通,再加。每幕 5~18 秒为宜,总长控制在 60~90 秒(需要更短用 make_2x.py 出快进版)。
